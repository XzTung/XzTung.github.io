Conda 虚拟环境管理常用命令

创建环境

目标 命令
创建名为 myenv 的环境 conda create -n myenv python=3.10
创建环境并指定安装包 conda create -n myenv python=3.10 numpy pandas matplotlib
创建并指定路径（不命名） conda create -p D:\project\envs\myenv python=3.10
创建环境时自动确认 在命令最后加 -y 选项，如：conda create -n myenv python=3.10 -y

激活 / 退出环境

操作 命令
激活命名环境 conda activate myenv
激活路径创建的环境 conda activate D:\project\envs\myenv
退出当前环境 conda deactivate

删除环境

操作 命令
删除命名环境 conda remove -n myenv --all
删除路径创建的环境 conda remove -p D:\project\envs\myenv --all

查看环境

操作 命令
查看所有环境 conda env list 或 conda info --envs
当前环境名为空？ 用 conda env list 可看到路径式环境，无名称的会显示路径

包管理

操作 命令
查看已安装包 conda list
查看其他环境包 conda list -n myenv
安装包（当前环境） conda install numpy pandas
指定频道安装 conda install -c conda-forge opencv
升级包 conda update numpy
升级 Conda 本体 conda update conda
删除包 conda remove numpy
查看包的依赖 conda info numpy
查找包 conda list | findstr torch

环境导入 / 导出（复现依赖）

操作 命令
导出环境（含版本依赖） conda env export > environment.yml
导入环境 conda env create -f environment.yml
导出为完整安装文件 conda list --explicit > env.txt
从完整包列表还原 conda create --name myenv --file env.txt

Conda 配置管理

操作 命令
添加默认频道 conda config --add channels conda-forge
设置频道优先级严格 conda config --set channel_priority strict
查看当前配置 conda config --show
重置配置为默认 conda config --remove-key channels

安装对应的工具包

对应一个项目的依赖，常在项目的根目录下寻找
常见的依赖文件名有：

  • requirements.txt（适用于 pip）
  • environment.yml（适用于 conda）
  • pyproject.toml（现代项目规范）
  • setup.py（如果是打包的 Python 包）
  • .ipynb 文件中的导入语句（没有专门的依赖文件时）
